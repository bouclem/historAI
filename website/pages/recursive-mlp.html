Recursive MLP

📅 1991 👤 historAI Project 🏷️ Divergence — Worse

Overview

The Recursive MLP is a divergence architecture — an MLP where the hidden layer feeds its own previous state back into itself, creating a recurrent loop. This is the natural next question after the MLP: "what if the hidden layer remembers?"

The architecture adds a recurrent weight matrix Wh alongside the standard input-to-hidden weights W1:

hidden_t = sigmoid(X_t · W1 + hidden_{t-1} · Wh + b1)

output_t = sigmoid(hidden_t · W2 + b2)

Training uses Backpropagation Through Time (BPTT) — the network is unrolled across time steps, and gradients flow backward through the recurrent connections. This is the same algorithm used to train RNNs today.

In real history, recurrent neural networks were developed independently by Rumelhart, Hinton, and Williams (1986) and by Elman (1990) with the Simple Recurrent Network. A "Recursive MLP" in 1991 sits at the intersection of these ideas — an MLP with self-recurrent hidden units.

Benchmark results: The Recursive MLP performs worse than the standard MLP and even the simple perceptron. On linearly separable data, it achieves only 50% accuracy (vs 100% for both perceptron and MLP). The recurrent connections cause vanishing gradients through the BPTT loop — the sigmoid derivative shrinks at each time step, making it difficult for gradients to flow. The network gets stuck predicting a single class. On high-dimensional data (10 features), it nearly matches at 99.7%, but on all other datasets it underperforms. This is the fundamental problem that would not be solved until LSTM (1997) and GRU (2014) introduced gating mechanisms to control information flow.
