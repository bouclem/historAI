Multi-Threshold Perceptron

📅 1958 👤 historAI Project 🏷️ Divergence — Worse

Overview

The Multi-Threshold Perceptron is a divergence architecture — a model that could have existed in 1958, right after Rosenblatt's perceptron, as a natural extension to multiple thresholds.

The standard perceptron uses a single threshold: fire if the weighted sum reaches θ. This model uses K thresholds θ₀ < θ₁ < ... < θₖ₋₁, dividing the real line into K+1 regions. The neuron fires in alternating regions — odd-indexed regions fire, even ones don't. This creates a multi-step activation function that can, in principle, learn non-linearly-separable patterns like XOR with a single neuron.

The learning rules:

w_i(t+1) = w_i(t) + r · (d_j − y_j) · x_j,i

θ_nearest += r · sign(z − θ_nearest) · |d_j − y_j|

When the neuron misclassifies, weights update as in the standard perceptron, and the threshold nearest to the current linear output moves toward it — shrinking the region that caused the error and expanding the adjacent region that would correct it.

Benchmark results: The multi-threshold approach performs worse than the standard perceptron. On linearly separable datasets, the alternating firing regions break the simple linear boundary — the neuron cannot settle on a single threshold that separates the two classes. It achieves 100% accuracy only on high-dimensional data where the thresholds collapse to a single effective boundary. On non-linearly separable datasets (concentric circles, XOR), it performs no better than the standard perceptron — both fail.
